3. Entrega 1: PROPUESTA INICIAL

1. Título propuesto

Plataforma Geoespacial Escalable para la Monitorización y Procesamiento de Datos Satelitales (Copernicus Sentinel-2): Caso de Uso Aplicado a los Embalses Catalanes.

En desarrollo: https://sentinel.tonilogar.com/

2. Objetivos del TFB

  • 2.1. Objetivo general: Desarrollar un pipeline geoespacial integral (Web y QGIS) que permita extraer, almacenar, procesar (ML) y visualizar interactiva y dinámicamente datos satelitales Sentinel-2.
  • 2.2. Objetivos específicos:
    • Integrar la API de Copernicus para visualización on the fly.
    • Diseñar una arquitectura de base de datos basada en Cloud Optimized GeoTIFF (COG) y PMTiles.
    • Entrenar un modelo de Machine Learning con series temporales para calcular la lámina de agua de 5 embalses catalanes.
    • Implementar un modelo de roles y pasarela de pago para la persistencia de datos y ejecución de algoritmos backend.

3. Breve justificación de la propuesta

  • Problema que resuelve (Visión Global y Proof of Concept): Si bien existen sensores físicos de alta precisión en zonas controladas, el verdadero valor de este TFB es crear una herramienta de bajo coste y alta escalabilidad. Los embalses catalanes se utilizarán como Laboratorio de Pruebas (Proof of Concept), aprovechando los datos exactos de la ACA (Ground Truth) para validar matemáticamente el modelo. Una vez validado, el sistema podrá desplegarse en zonas remotas del planeta (países en vías de desarrollo, zonas de conflicto) donde no existe infraestructura de medición, permitiendo a ONGs y gobiernos monitorizar reservas hídricas usando únicamente datos espaciales abiertos.
  • Extensibilidad del Pipeline: Más allá del caso del agua, la arquitectura está diseñada para demostrar que el flujo de datos (pipeline) funciona de principio a fin. Cambiando únicamente el modelo de Machine Learning en el backend, esta misma plataforma es capaz de resolver otros problemas globales utilizando la constelación Copernicus: control de la deforestación, seguimiento de plagas agrícolas, expansión de zonas urbanas, etc.
  • Códigos UNESCO sugeridos: 1203 (Ciencia de los ordenadores), 2505 (Geografía), 3329 (Planificación urbana y regional).
  • ODS: ODS 6 (Agua limpia y saneamiento) y ODS 13 (Acción por el clima).

5. Alcance previo

  • Alcance positivo (In-scope): Desarrollo web interactivo, Plugin QGIS, Extracción Copernicus Sentinel-2, Modelo ML para 5 embalses catalanes, visualización web con COG/PMTiles, Sistema de Auth/Pagos básico.
  • Alcance negativo (Out-of-scope): No se desarrollarán modelos predictivos para otras zonas del mundo ni para otros usos que no sean la detección de cuerpos de agua. No se procesarán imágenes de radar (Sentinel-1).

6. Competencias que desarrollar

  • CE2 (Estructurar información): Nivel 4
  • CE3 (Diseño BD y almacenamiento - COG/PMTiles): Nivel 5
  • CE4 (Machine learning y toma de decisiones): Nivel 4
  • CE5 (Modelización estadística): Nivel 3
  • CE8 (Gestión integral del proyecto): Nivel 4
  • CE10 (Legislación y normativa): Nivel 2
  • CE11 (Visualización interactiva): Nivel 5

8. Análisis de riesgos

  • Riesgo 1: Limitación de cuotas de la API Copernicus.
    Mitigación: Implementar caché local de teselas y aplicar limitación de peticiones por usuario.
  • Riesgo 2: Falta de precisión del modelo ML por nubes.
    Mitigación: Aplicar filtros de nubosidad estrictos en el pipeline de extracción antes de la inferencia.
